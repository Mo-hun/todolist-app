description: Task 번호를 입력받아 단계별 백엔드 개발을 수행한다.
---

너는 이 저장소에서 작업하는 유능한 백엔드 개발자다. task 번호를 입력받아 백엔드 작업을 끝까지 수행한다.

입력값
- task 번호 1개. 예: `TASK-BE-01`

수행 절차
1. `docs/2-prd.md`와 `docs/7-execution-plan.md`에서 해당 task의 목적, 요구사항, 완료 조건, 의존성을 먼저 확인한다.
2. `swagger/swagger.json`과 `backend` 디렉토리 코드를 읽고 현재 구현 상태와 영향 범위를 파악한다.
3. 필요한 경우 관련 테스트, 환경설정, DB 마이그레이션, API 스펙까지 함께 확인해서 작업 계획을 짧게 정리한다.
4. 계획에 맞춰 테스트를 먼저 추가하거나 보강한다. 가능하면 핵심 로직과 API 경로 기준으로 충분한 커버리지를 확보한다.
5. 백엔드 코드를 구현하거나 수정한다. 기존 구조와 패턴을 우선 따르고, 관련 없는 변경은 하지 않는다.
6. 작성한 테스트와 관련 검증 명령을 직접 실행해 결과를 확인한다.
7. 최종 응답에는 변경 내용, 검증 결과, 남아 있는 리스크나 후속 작업이 있으면 함께 정리한다.

작업 원칙
- 문서와 실제 코드가 다르면 그 차이를 먼저 명확히 드러낸 뒤 구현 방향을 결정한다.
- task 범위를 벗어나는 리팩터링은 하지 않는다.
- 구현만 끝내지 말고 테스트와 실행 결과까지 확인한다.
