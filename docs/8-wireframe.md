## 1. 공통 컴포넌트 정의

### 1.1 Button

```
[Primary]                [Secondary]              [Danger]
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│    버튼 텍스트    │     │    버튼 텍스트    │     │    버튼 텍스트    │
└──────────────────┘     └──────────────────┘     └──────────────────┘
 bg-blue-600              bg-white/border           bg-red-600
 text-white               text-gray-700             text-white

[Disabled]               [Loading]
┌──────────────────┐     ┌──────────────────┐
│    버튼 텍스트    │     │  ◌  처리 중...   │
└──────────────────┘     └──────────────────┘
 opacity-50               Spinner + 텍스트
 cursor-not-allowed        cursor-not-allowed
```

변형 설명:
- Primary: 핵심 액션 (저장, 로그인, 생성)에 사용. bg-blue-600 배경
- Secondary: 보조 액션 (취소, 뒤로가기)에 사용. 테두리 있는 흰 배경
- Danger: 파괴적 액션 (삭제, 탈퇴)에 사용. bg-red-600 배경
- Disabled: 입력값 미충족 시 비활성. opacity-50 처리
- Loading: API 요청 중. Spinner 아이콘 + "처리 중..." 텍스트, 클릭 불가

### 1.2 InputField

```
[기본 상태]
┌─────────────────────────────────────────┐
│ 레이블 텍스트                            │
│ ┌─────────────────────────────────────┐ │
│ │ placeholder 텍스트                  │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘

[포커스 상태]
┌─────────────────────────────────────────┐
│ 레이블 텍스트                            │
│ ┌─────────────────────────────────────┐ │
│ │ 입력 중인 텍스트                    │ │  ← ring-2 ring-blue-500
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘

[에러 상태]
┌─────────────────────────────────────────┐
│ 레이블 텍스트                            │
│ ┌─────────────────────────────────────┐ │
│ │ 잘못된 값                           │ │  ← border-red-500
│ └─────────────────────────────────────┘ │

```

상태 설명:
- 기본: border-gray-300, rounded-md, 레이블은 필드 상단에 표시
- 포커스: ring-2 ring-blue-500으로 포커스 링 강조
- 에러: border-red-500, 하단 에러 메시지 text-red-600으로 표시
- 비밀번호 필드: 우측에 표시/숨김 토글 아이콘 제공

- [X] 버튼: 모달 닫힘

- 모바일에서 전체 화면 너비(w-full) 적용, 데스크탑에서 max-w-md

### 1.4 Badge

```
[카테고리 배지]         [상태 배지]
┌───────────┐          ┌───────────┐  ┌───────────┐  ┌───────────┐  ┌───────────┐
│  #업무    │          │  진행중   │  │  기한임박 │  │  기한초과 │  │   완료    │
└───────────┘          └───────────┘  └───────────┘  └───────────┘  └───────────┘
 bg-gray-100            bg-gray-100    bg-orange-100   bg-red-100    bg-green-100
 text-gray-700          text-gray-700  text-orange-700 text-red-700  text-green-700
```

배지 용도:
- 카테고리 배지: TodoItem에서 카테고리 이름을 컴팩트하게 표시
- 상태 배지: 할일 상태를 텍스트로 보조 표시 (배경색과 함께 사용)
- rounded-full 처리로 pill 형태, text-xs 크기

### 1.5 Spinner

```
  ◌     ←  animate-spin
  
  전체 로딩: 페이지 중앙 배치, w-8 h-8
  버튼 로딩: 버튼 내부 좌측, w-4 h-4
```

- border-4 border-gray-200 border-t-blue-600 rounded-full animate-spin
- 페이지 단위 로딩: 화면 중앙에 단독 표시
- 버튼 단위 로딩: 버튼 텍스트 좌측에 소형으로 표시

### 2.3 UX 설명

- 페이지 진입: 비인증 전용 라우트(PublicRoute). 이미 로그인된 사용자가 접근하면 메인 페이지(/)로 자동 리다이렉트
- 이메일 필드: type="email" 입력. 포커스 시 ring-2 ring-blue-500 강조. 형식 오류 시 필드 하단에 text-red-600 에러 메시지 즉시 표시
- 비밀번호 필드: type="password" 입력. 우측 "보기" 토글로 평문 확인 가능
- 에러 상태: 로그인 실패 시(401) "이메일 또는 비밀번호가 올바르지 않습니다." 메시지를 폼 내부에 표시. 보안상 어느 쪽 오류인지 구분하지 않음
- 로그인 버튼: 기본 Primary 스타일. 요청 중 Loading 상태로 전환(Spinner + "로그인 중..."), 중복 클릭 방지
- 로그인 성공: JWT를 authStore에 저장 후 메인 페이지(/)로 이동
- "비밀번호를 잊으셨나요?": 비밀번호 재설정 플로우 링크 (Should 우선순위, Phase 1 포함)
- "회원가입 하러가기": /register 페이지로 이동하는 텍스트 링크
- 데스크탑 레이아웃: 좌측 서비스 소개 패널 + 우측 로그인 카드 형태의 2단 분할 구성

- 페이지 진입: 비인증 전용 라우트(PublicRoute). 이미 로그인된 사용자는 메인 페이지(/)로 자동 리다이렉트

- 비밀번호 필드: 최소 8자 조건. 조건 미충족 시 필드 하단에 안내 문구 표시
- 비밀번호 확인 필드: 비밀번호 필드 값과 실시간 비교. 값이 다를 경우 "비밀번호가 일치하지 않습니다." 에러 표시. 서버 요청 이전 클라이언트 단 검증으로 불필요한 API 호출 방지
- 회원가입 버튼: 세 필드 모두 유효한 경우에만 활성화. 요청 중 Loading 상태로 전환
- 중복 이메일(409 응답): "이미 사용 중인 이메일입니다." 에러를 이메일 필드 하단에 표시
- 회원가입 성공: /login 페이지로 리다이렉트. 별도 안내 메시지 없이 자연스러운 이동
- "로그인 하러가기": /login으로 이동하는 텍스트 링크
- 데스크탑 레이아웃: 로그인 페이지와 동일한 2단 분할 구조로 일관성 유지

- 제목 필드: 필수 항목(*). 빈 값 저장 시도 시 "제목은 필수입니다." 에러 표시
- 설명 필드: 선택 항목. textarea 형태, 여러 줄 입력 가능. 미입력 시 null로 저장
- 카테고리 드롭다운: 선택 항목. useGetCategories로 로드한 사용자 카테고리 목록 표시. 첫 옵션 "카테고리 선택 (선택)"으로 미선택 기본값 제공. 미선택 시 category_id = null
- 마감일 필드: 선택 항목(BR-DATA-04). input type="date" 사용. 미입력 시 due_date = null로 저장. null이면 기한초과 판별 제외
- [취소] 버튼: Secondary 스타일. 입력 내용 저장 없이 모달 닫힘
- [저장]/[수정 저장] 버튼: Primary 스타일. 요청 중 Loading 상태. 성공 시 ['todos'] 쿼리 무효화 → 목록 즉시 갱신
- 수정 모드 진입: 기존 title, description, category_id, due_date 값이 각 필드에 미리 채워진 상태로 모달 오픈

### 6.2 카테고리 생성/수정 Modal

**[생성 모드]**
```
┌─────────────────────────────────────────┐
│ 새 카테고리 추가                    [X] │
├─────────────────────────────────────────┤
│                                         │
│  카테고리 이름 *                        │
│  ┌───────────────────────────────────┐  │
│  │ 카테고리 이름을 입력하세요        │  │
│  └───────────────────────────────────┘  │
│  ! 에러 메시지 영역                     │
│                                         │
├─────────────────────────────────────────┤
│              [취소]      [추가]         │
└─────────────────────────────────────────┘
```

**[수정 모드]**
```
┌─────────────────────────────────────────┐
│ 카테고리 수정                       [X] │
├─────────────────────────────────────────┤
│                                         │
│  카테고리 이름 *                        │
│  ┌───────────────────────────────────┐  │
│  │ 업무                              │  │  ← 기존 이름 미리 채워짐
│  └───────────────────────────────────┘  │
│  ! 에러 메시지 영역                     │
│                                         │
├─────────────────────────────────────────┤
│              [취소]   [수정 저장]       │
└─────────────────────────────────────────┘
```

UX 설명:
- 단일 입력 필드로 간결한 구성. 카테고리는 이름만 관리
- 이름 필드: 필수 항목(*). 빈 값 저장 시 "카테고리 이름을 입력해주세요." 에러 표시
- 중복 이름 서버 에러 시 "동일한 이름의 카테고리가 이미 있습니다." 에러 표시
- 수정 모드: 기존 이름이 입력 필드에 채워진 상태로 오픈. 자동 포커스 및 텍스트 전체 선택으로 즉시 수정 가능
- [추가]/[수정 저장] 버튼: Primary 스타일. 성공 시 ['categories'] 쿼리 무효화 → 목록 즉시 갱신
- [취소]: Secondary 스타일. 변경 사항 없이 모달 닫힘
- 모달의 작은 크기(max-w-sm)로 모바일 화면 내 자연스러운 표시

### 6.3 삭제 확인 Modal

```
┌─────────────────────────────────────────┐
│ 할일 삭제                           [X] │

│                                         │
│  정말 이 할일을 삭제하시겠습니까?       │
│                                         │

│                                         │

│                                         │
├─────────────────────────────────────────┤
│              [취소]      [삭제]         │
└─────────────────────────────────────────┘
```

**[카테고리 삭제 확인 — 할일이 있는 경우]**
```
┌─────────────────────────────────────────┐
│ 카테고리 삭제                       [X] │
├─────────────────────────────────────────┤
│                                         │
│  "업무" 카테고리를 삭제하시겠습니까?   │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │ ⚠  이 카테고리에 속한 5개의       │  │
│  │    할일이 '미분류'로 이동됩니다.  │  │
│  │    할일 데이터는 삭제되지 않습니다│  │
│  └───────────────────────────────────┘  │
│                                         │
├─────────────────────────────────────────┤
│              [취소]      [삭제]         │
└─────────────────────────────────────────┘
```

**[카테고리 삭제 확인 — 할일이 없는 경우]**
```
┌─────────────────────────────────────────┐
│ 카테고리 삭제                       [X] │
├─────────────────────────────────────────┤
│                                         │
│  "학습" 카테고리를 삭제하시겠습니까?   │
│                                         │
│  삭제된 카테고리는 복구할 수 없습니다. │
│                                         │
├─────────────────────────────────────────┤
│              [취소]      [삭제]         │
└─────────────────────────────────────────┘
```

UX 설명:
- 삭제는 파괴적 액션이므로 반드시 확인 단계 제공. 단순 아이콘 클릭만으로 즉시 삭제되지 않음
- 삭제 대상 이름을 모달 본문에 명시적으로 표시하여 사용자가 어떤 항목을 삭제하는지 명확히 인지
- [삭제] 버튼: Danger 스타일(bg-red-600). 시각적으로 파괴적 액션임을 강조
- [취소] 버튼: Secondary 스타일. 모달 닫힘, 데이터 변경 없음
- 카테고리 삭제 시 할일이 있는 경우: 경고 박스(bg-orange-50, border-orange-300)로 BR-DATA-03 규칙 사용자에게 사전 안내
- 요청 중 [삭제] 버튼 Loading 상태 전환. 성공 시 해당 쿼리 무효화 → 목록 즉시 갱신
- 오버레이 클릭 또는 ESC 키: 취소와 동일하게 처리. 실수로 인한 삭제 방지
