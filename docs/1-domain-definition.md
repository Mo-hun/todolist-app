## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | TodoList |
| 유형 | 개인용 할일 관리 웹 애플리케이션 |
| 목적 | 카테고리 및 종료일 기반의 체계적인 개인 할일 관리 |
| 핵심 KPI | 할일 등록 후 완료까지의 평균 리드타임 감소, 기한 초과 할일 발생률 20% 감소 목표 |
| 대상 사용자 | 개인 할일을 체계적으로 관리하려는 일반 사용자 |

---

## 2. 문제 정의

- 카테고리 분류를 지원하는 적절한 할일 관리 앱 부재
- 할일의 종료일(마감일)을 명확하게 표현하고 상태별로 구분하는 기능 부재
- 개인화된 카테고리를 자유롭게 구성할 수 없는 기존 앱의 한계

---

## 3. 핵심 도메인 개념 (용어 정의)

| 용어 | 정의 |
|------|------|
| 사용자 (User) | 회원 가입 후 인증을 통해 앱을 사용하는 주체 |
| 할일 (Todo) | 사용자가 등록한 수행해야 할 작업 단위 |
| 카테고리 (Category) | 사용자가 직접 정의하여 할일을 분류하는 그룹 |
| 종료일 (Due Date) | 할일의 완료 목표 일시 |
| 완료 상태 (Completed) | 사용자가 할일을 직접 완료 처리한 상태 |
| 기한 초과 (Overdue) | 종료일이 현재 일시를 경과했으나 미완료인 상태 |
| 진행 중 (In Progress) | 종료일이 남아 있고 미완료인 상태 |

## 4. 핵심 기능 요구사항

### 4.1 인증 및 계정 (AUTH)
- **FR-AUTH-01**: 이메일/비밀번호 기반 회원 가입 및 로그인
- **FR-AUTH-02**: 비인증 사용자는 로그인/회원가입 페이지 외 모든 기능 접근 불가 (로그인 페이지로 리다이렉트)
- **FR-AUTH-03**: 비밀번호 찾기 및 재설정 (이메일 인증 기반)
- **FR-AUTH-04**: 회원 탈퇴 (본인 확인 후 처리)

### 4.2 할일 관리 (TODO)
- **FR-TODO-01**: 할일 생성 · 조회 · 수정 · 삭제 (CRUD)
- **FR-TODO-02**: 할일 작성 시 카테고리 및 종료일 지정 (Optional)
- **FR-TODO-03**: 할일 완료 처리 및 완료 취소 (상태 전환)

### 4.3 카테고리 관리 (CAT)
- **FR-CAT-01**: 카테고리 생성 · 수정 · 삭제
- **FR-CAT-02**: 카테고리별 할일 필터링 및 목록 조회

### 4.4 대시보드 및 표시 (VIEW)
- **FR-VIEW-01**: 완료 / 기한 초과 / 진행 중 할일의 시각적 구분 표시
- **FR-VIEW-02**: 기한 임박 할일(24시간 이내)에 대한 강조 표시

## 5. 도메인 규칙 (비즈니스 규칙)

### 5.1 권한 및 보안 규칙
| ID | 규칙 | 관련 FR |
|---|---|---|
| BR-AUTH-01 | 모든 기능은 로그인 인증 후에만 사용 가능하다. | FR-AUTH-02 |
| BR-AUTH-02 | 이메일은 사용자 식별의 고유 키이며 중복 가입을 허용하지 않는다. | FR-AUTH-01 |
| BR-AUTH-03 | 사용자는 자신이 생성한 데이터(할일, 카테고리)만 접근 가능하다. | ALL |

### 5.2 데이터 관리 규칙
| ID | 규칙 | 관련 FR |
|---|---|---|
| BR-DATA-01 | 할일의 기한 초과 여부는 종료일과 현재 일시를 비교하여 자동 판별한다. | FR-VIEW-01 |
| BR-DATA-02 | 완료 처리된 할일은 기한 초과 상태로 분류되지 않는다. | FR-TODO-03 |
| BR-DATA-03 | 카테고리 삭제 시, 속한 할일은 '미분류(null)' 상태로 보존된다. | FR-CAT-01 |
| BR-DATA-04 | 종료일은 선택 항목이며, 미입력 시 기한 초과 판별 대상에서 제외된다. | FR-TODO-02 |
| BR-DATA-05 | 회원 탈퇴 시 해당 사용자의 모든 데이터는 즉시 삭제(Hard Delete)된다. | FR-AUTH-04 |

## 6. 주요 도메인 객체

### User (사용자)
- `id` (UUID), `email` (String, Unique), `password` (String, Encrypted), `created_at` (DateTime)

### Category (카테고리)
- `id` (UUID), `user_id` (UUID, FK), `name` (String), `created_at` (DateTime)

### Todo (할일)
- `id` (UUID), `user_id` (UUID, FK), `category_id` (UUID, FK, Nullable), `title` (String), `description` (Text, Nullable), `due_date` (DateTime, Nullable), `is_completed` (Boolean), `created_at` (DateTime), `updated_at` (DateTime)

---

## 7. 비기능적 요구사항 (NFR)

- **NFR-SEC-01**: 모든 비밀번호는 Argon2 또는 bcrypt 등 강력한 해시 알고리즘으로 암호화하여 저장한다.
- **NFR-PER-01**: 할일 목록 조회 API 응답 속도는 200ms 이내여야 한다.
- **NFR-USA-01**: 모바일 환경에서도 할일 확인이 용이하도록 반응형 웹 디자인을 적용한다.

## 8. 요구사항 추적 매트릭스 (RTM)

| 기능 ID | 관련 비즈니스 규칙 | 검증 방법 |
|---|---|---|
| FR-AUTH-01 | BR-AUTH-02 | 회원가입 테스트 (중복 체크 포함) |
| FR-TODO-01 | BR-AUTH-03 | 타인 할일 접근 차단 테스트 |
| FR-TODO-03 | BR-DATA-02 | 완료 처리 후 기한 초과 로직 제외 확인 |
| FR-CAT-01 | BR-DATA-03 | 카테고리 삭제 후 할일 보존 상태 확인 |
| FR-AUTH-04 | BR-DATA-05 | 탈퇴 후 DB 내 데이터 완전 삭제 확인 |
